## スケジュール・進捗管理

週次の進捗は [learning_schedule.md](learning_schedule.md) で管理します。

### 状態の凡例

| 記号 | 意味 |
|---|---|
| ⬜ 未着手 | まだ開始していない |
| 🔵 進行中 | 学習を進めている |
| ✅ 完了 | 学習と確認テスト完了 |
| ⏸️ 保留 | 一時中断中 |

## Claude Code との連携ルール

### 1. 学習時間の記録

以下のフォーマットで宣言すると、経過時間を計測してスケジュールの実績時間・状態を自動更新します。

| 宣言 | タイミング |
|---|---|
| `学習開始: W1 光と色の基礎` | 学習を始めるとき |
| `学習終了: W1 光と色の基礎` | 学習を終えるとき |
| `学習中断: W1 光と色の基礎` | 途中で中断するとき |
| `学習再開: W1 光と色の基礎` | 再開するとき |

**中断・再開を繰り返した場合も、経過時間を累計して実績に反映します。**

例：
```
学習開始: W1 光と色の基礎
  → 1時間後
学習中断: W1 光と色の基礎   ← 1h 記録

  （翌日）
学習再開: W1 光と色の基礎
  → 30分後
学習終了: W1 光と色の基礎   ← 累計 1.5h でスケジュール更新・状態を✅完了
```

---

### 2. 確認テストの回答

各教材末尾の確認テスト（10問）に回答する際は、以下のフォーマットで送信してください。
Claude Code が解答ファイルと照合して正誤・スコア・解説をフィードバックします。

```
テスト回答: 3級 光と色の基礎

Q1: B
Q2: 可視光線
Q3: 演色性
...
```

- `テスト回答:` の後に「3級 or 2級」とトピック名を記載する
- 全問でなく一部（例: Q1〜Q5）だけでもOK

---

### 3. 模擬試験の受験

模擬試験を受ける際は以下のフォーマットで送信してください。
Claude Code が問題（30問）を出題し、回答後に正誤・スコア・解説を返します。

```
模擬試験: 3級 第1回
```

---

### 4. フェーズ完了までの流れ

```
1. 学習終了: Wx ○○           ← 時間記録・スケジュール更新
2. テスト回答: 3級 or 2級 ○○ ← 確認テスト（10問・8割以上で合格）
3. 模擬試験: 3級 or 2級 第X回 ← 模擬試験（30問・7割以上で次へ）
4. → 次のトピックへ
```
